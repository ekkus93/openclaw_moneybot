# submission_package_builder

## Purpose

Turn an approved opportunity into a concrete, traceable submission package so execution stays bounded and complete.

## Inputs

- approved opportunity and rules text
- policy, TOS, and budget references
- archived source evidence

## Outputs

- deterministic checklist
- required fields and artifacts
- submission URL and deadline when available
- unresolved-item list
- archived package snapshot

## Fail-closed behavior

- missing or conflicting instructions block or require review
- the skill never invents missing submission content

## Example

Rules that list required fields, screenshots, and a submission URL become a stable checklist that downstream execution can follow.

## Non-goals

- auto-submitting forms
- fabricating deliverables
